# ADR: 数据源切换（SQLite / PostgreSQL）

日期：2026-03-10
状态：已接受

## 背景
当前应用使用 SQLite 作为本地存储，并在 SQLite 模式下启用 Git 同步。用户希望增加本地 PostgreSQL 作为可选数据源，并通过配置开关切换。切换时不迁移旧数据源数据。

## 决策
- 引入数据源路由层，通过配置在 SQLite 与 PostgreSQL 之间切换。
- 仅 SQLite 模式启用 Git 同步；PostgreSQL 模式禁用同步入口。
- 切换数据源时不迁移旧数据源数据，切换后以当前数据源为准。
- PostgreSQL 连接信息保存在本地配置文件中。

## 备选方案
1. 事件日志为唯一事实源，SQLite/PostgreSQL 均为派生读库。
   - 优点：一致性强、避免分叉。
   - 缺点：工程量大，超出当前需求。
2. 全量迁移并以 PostgreSQL 为唯一事实源。
   - 优点：单一数据库技术栈。
   - 缺点：破坏本地优先与 Git 同步机制，运维成本高。

## 影响
- 优点：实现成本低，满足“可切换数据源”需求。
- 缺点：数据会分叉；切换后旧数据不可见，需要明确提示。
- 风险：PG 连接失败导致无法切换；配置文件包含明文密码。

## 缓解
- 切换时强提示“不迁移数据”，并提供“重新导入”入口。
- 连接失败时保持当前数据源不变。
- 文档中明确配置文件敏感信息风险。
