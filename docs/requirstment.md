我想构建一个软件，它的主要作用是做浏览器书签的同步。

他可以支持所有的浏览器，浏览器当前的书签和后续写入的书签都会同步到这个应用里，这个应用有一个专业的界面可以展示当前的书签。他支持手动分组、给书签添加标签、按照 host 进行分组等你认为需要的功能。

通过github仓库进行增量同步不同设备之间的书签，同一个url只会有一条记录。




方案二：Git 存储引入 CQRS / Event Sourcing (事件溯源) 机制
不要在 Git 里直接存最终的“树状大JSON”。

在 Git 中，把每一次书签变动视为一条“追加日志（Append-only log）”，例如：add_bookmark_abc.json, remove_bookmark_xyz.json。
无冲突设计：由于两台设备产生的永远是新文件（基于时间戳或 UUID 命名），因此用 Git Pull/Push 时永远不会发生文件行级冲突。
本地拉取这些增量日志后，在本地的 SQLite 中回放这些事件（合并、去重），重构出最终的书签树。


方案三：引入 AI 辅助的 URL 泛化与元数据补全
智能清洗：在保存书签时，剥离已知的追踪参数，提取真正的 Canonical URL 进行唯一性校验。
为唯一的 URL 记录引入多对多关系：一个 URL 实体可以对应多个“目录分组”或“标签”。
可以在抓取书签时通过后台悄悄下载该页面的 Title、Description 和 Icon，甚至本地跑一个极轻的 NLP 模型，自动给死板的书签打上如 前端、Rust、娱乐 的 AI 标签。
